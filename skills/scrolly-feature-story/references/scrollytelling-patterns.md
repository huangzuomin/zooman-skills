# Scrollytelling Patterns

Use these patterns to map content into a scrolling story. Pick only the patterns that fit the content; do not stack effects for spectacle.

## Sticky Graphic + Step Text

Best for product systems, maps, data charts, process diagrams, and capability matrices.

Desktop:

- One side is a sticky visual.
- The opposite side contains sequential text steps.
- Each step changes highlights, labels, cards, or diagram state.

Mobile:

- Convert to a normal reading flow.
- Place the relevant visual near each step.
- Avoid long pinned sections.

## Full-Bleed Chapter

Best for hero openings, section breaks, emotional pivots, and closing statements.

Rules:

- Use one dominant image, video still, generated visual, or strong typographic composition.
- Keep text short.
- Ensure contrast with overlays or careful placement.
- Avoid decorative dark blurred stock-like backgrounds when the subject matters.

## Editorial Longform

Best for manifestos, context, analysis, and reflective sections.

Components:

- Large display heading.
- Narrow body column.
- Pull quote.
- Side note or metadata rail.
- Occasional full-width image.

Avoid making it look like raw Markdown.

## Capability Matrix

Best for service manuals, product systems, and “what we can provide” sections.

Flow:

1. Show the whole system.
2. Highlight one capability group at a time.
3. Expand product/service details.
4. Recombine into service packages.

Use when the content needs synthesis rather than simple listing.

## Case Gallery

Best for proof, examples, project portfolios, and service credibility.

Desktop:

- Use filterable or grouped cards if there are many cases.
- Use type labels and one-line value statements.

Mobile:

- Prefer a vertical list or compact cards.
- Keep the value statement visible without hover.

## Package Cards

Best for bundles, plans, solution sets, and customer scenarios.

Each card should show:

- Customer profile.
- Included services.
- Best-fit scenario.
- Primary outcome.

Interaction:

- Expand details on click, not hover-only.
- Preserve keyboard access.

## Data Annotation

Best for charts, timelines, public-opinion data, performance results, and trend explanation.

Pattern:

- Keep the chart stable.
- Scroll text highlights one area, time range, or metric.
- Use annotations and labels instead of chart-type switching unless the switch has meaning.

## Dynamic Split Screen

Best for contrast, before/after, dual perspectives, or parallel timelines.

Use sparingly. On mobile, stack the two narratives clearly and label them.

## Grid Break / Reform

Best for showing scattered abilities becoming a product system, or case fragments becoming a portfolio.

Use transforms and opacity. Avoid layout-thrashing animations.

## Object Assembly

Best for explaining how a system, product package, or process is built.

Animate parts into place only when the construction itself communicates meaning.
