# Template Selection

Use this reference before designing. A scrollytelling page needs a named composition model. Without a template family, the output often becomes a generic grid of cards.

## Selection Protocol

Before implementation, write a short design decision:

```text
Template family:
Primary visual direction:
Secondary influence:
Audience:
Why this fits:
Repeated motifs:
Forbidden motifs:
Required assets:
Mobile adaptation:
```

Do not begin markup until this decision is clear.

## Template Families

### Civic Editorial Atlas

Use for city, public-service, local media, government-adjacent, and civic storytelling.

Best for:

- City service platforms.
- Public communication manuals.
- Regional media transformation stories.
- County/city collaboration maps.

Composition motifs:

- Map/grid canvas.
- Regional labels and coordinates.
- Large editorial headlines.
- Documentary image strips.
- Institutional captions.
- Section dividers that feel like map sheets or field notes.

Asset needs:

- City skyline or aerial.
- Map or district outline.
- Public-service scenes.
- Newsroom/workflow images.
- Data overlays.

Avoid:

- Generic dark tech dashboards.
- SaaS feature blocks.
- Decorative map dots without location meaning.

### Newsroom Product Bureau

Use for media organizations turning editorial, platform, data, event, and AI capabilities into products.

Best for:

- Product/service manuals from news organizations.
- Media business transformation pages.
- Internal-to-external capability packaging.

Composition motifs:

- Newsroom desk / assignment board.
- Product dossier pages.
- Editorial rails.
- Capability index.
- Case clippings.
- Service cards that feel like briefs, not SaaS cards.

Asset needs:

- Newsroom/editorial process visuals.
- Article/page screenshots.
- Platform operation screenshots.
- Case evidence snippets.
- Product taxonomy diagrams.

Avoid:

- Over-polished startup marketing style.
- Three-column “feature icon” sections.
- Unexplained abstract particles.

### Data Dossier

Use for evidence-heavy stories, monitoring reports, public opinion, trends, and dashboards.

Best for:

- Public opinion monitoring.
- Performance reports.
- Data products.
- Risk and trend explanation.

Composition motifs:

- Annotated charts.
- Dossier tabs.
- Highlighted evidence strips.
- Timeline rails.
- Chart + explanatory step text.

Asset needs:

- Charts.
- Tables.
- Map/data screenshots.
- Report pages.
- Alert/event examples.

Avoid:

- Fake dashboards.
- Decorative sparklines.
- Charts with no claim.

### Service Field Guide

Use for product manuals, service catalogs, customer-facing packages, and sales enablement.

Best for:

- Product/service manuals.
- Package comparison.
- Capability catalogs.
- Client scenario guides.

Composition motifs:

- Indexed sections.
- Field-guide labels.
- Scenario pages.
- Service package sheets.
- Capability cards with clear evidence.

Asset needs:

- Service workflow diagrams.
- Product screenshots.
- Case images.
- Package diagrams.
- Customer scenario illustrations.

Avoid:

- Every product as an identical card.
- Pricing-table aesthetics unless there are real prices.
- Hiding important package details behind hover-only UI.

### Campaign Feature

Use for activities, competitions, cultural events, launches, and calls for participation.

Best for:

- Event pages.
- Contest pages.
- Cultural campaign pages.
- Registration/participation stories.

Composition motifs:

- Strong full-bleed opening.
- Fast chapter rhythm.
- Participant path.
- Prize/details modules.
- CTA panels.

Asset needs:

- Event hero image or video.
- People/action scenes.
- Venue imagery.
- Registration or participation screenshots.

Avoid:

- Slow institutional rhythm.
- Excessive text density.
- CTA buried below long explanations.

## Template Application Rules

The chosen family must affect:

- Hero composition.
- Section rhythm.
- Type scale.
- Asset style.
- Card shape and density.
- Navigation style.
- Case presentation.
- Closing section.

If the same layout could fit any project after changing text, the template has not been applied strongly enough.

## Recommended For 温州新闻网产品手册

Preferred combination:

```text
Template family: Newsroom Product Bureau
Secondary influence: Civic Editorial Atlas
Tertiary pattern: Data Dossier for 舆情/AI/案例 sections
```

Why:

- The story is about a news organization turning internal capabilities into external products.
- It needs newsroom credibility, city service context, and product clarity.
- It should feel like a media group's strategic product dossier, not a generic government website or startup landing page.

Useful motifs:

- “产品档案” cards instead of generic feature cards.
- “能力索引” instead of plain icon grid.
- “案例剪报” instead of portfolio cards.
- “服务组合单” instead of pricing cards.
- “城市服务地图” as the opening visual.
